## 二、基础版本代码（含逐行说明）
### 代码文件：`app/api/v1/user.py`
```python
# 文件名：app/api/v1/user.py
# 功能：用户注册接口基础实现

# 1. 导入核心依赖：FastAPI核心模块、Pydantic数据校验、数据库连接
from fastapi import APIRouter, HTTPException  # 路由注册/HTTP异常处理
from pydantic import BaseModel, EmailStr      # 数据模型/邮箱格式校验
from app.db.mysql import db_session           # 自定义数据库会话（基础版）
from app.models.user import User              # 用户数据库模型（ORM）

# 2. 初始化路由：前缀/v1/user，标签用于接口文档分组
router = APIRouter(prefix="/v1/user", tags=["用户管理"])

# 3. 定义请求体模型：校验前端传入的注册参数
class UserRegisterRequest(BaseModel):
    username: str          # 用户名（基础校验：非空）
    email: EmailStr        # 邮箱（Pydantic内置邮箱格式校验）
    password: str          # 密码（基础版：仅校验非空）

# 4. 注册接口实现：POST请求，路径/register
@router.post("/register")
def user_register(req: UserRegisterRequest):
    # 4.1 校验用户是否已存在：通过邮箱查询
    existing_user = db_session.query(User).filter(User.email == req.email).first()
    if existing_user:
        # 4.2 抛出400异常：邮箱已注册
        raise HTTPException(status_code=400, detail="该邮箱已注册")
    
    # 4.3 创建新用户：密码明文存储（基础版缺陷）
    new_user = User(
        username=req.username,
        email=req.email,
        password=req.password
    )
    
    # 4.4 提交数据库事务
    db_session.add(new_user)
    db_session.commit()
    
    # 4.5 返回注册成功响应
    return {"code": 200, "msg": "注册成功", "data": {"user_id": new_user.id}}
```

### 逐行说明表
| 代码段                | 核心作用                                                                 |
|-----------------------|--------------------------------------------------------------------------|
| `from fastapi import...` | 导入FastAPI路由、异常处理模块，Pydantic校验模块，自定义数据库/模型依赖    |
| `router = APIRouter(...)` | 初始化路由实例，指定接口前缀和标签，实现路由模块化拆分                   |
| `class UserRegisterRequest...` | 定义请求体校验模型，基础版仅校验字段类型（用户名非空、邮箱格式合法）     |
| `@router.post("/register")` | 装饰器声明POST接口，路径为`/v1/user/register`                           |
| `existing_user = db_session.query...` | 通过ORM查询邮箱是否已注册，基础版未做参数过滤（低风险）|
| `raise HTTPException(...)` | 邮箱已注册时抛出400异常，返回自定义提示信息                             |
| `new_user = User(...)` | 创建用户ORM实例，**核心缺陷**：密码明文存储，存在泄露风险                |
| `db_session.add(new_user)` | 将新用户实例加入数据库会话                                               |
| `db_session.commit()` | 提交事务，基础版未做异常捕获，失败会导致事务未回滚                       |
| `return {...}` | 返回注册成功响应，基础版未约束响应格式                                   |

### 目录设计说明
| 目录层级       | 设计目的                                                                 |
|----------------|--------------------------------------------------------------------------|
| `api/`         | 仅处理HTTP协议相关逻辑（路由、参数接收、响应返回），不包含业务逻辑       |
| `core/`        | 存放全局通用能力，避免重复编码，统一配置/异常/工具类管理                 |
| `db/`          | 封装数据库连接逻辑，对外提供标准化会话/连接实例，屏蔽底层数据库差异       |
| `models/`      | 纯数据库ORM模型，仅定义字段、索引、关联关系，无业务逻辑                 |
| `schemas/`     | 独立的请求/响应校验层，与数据库模型解耦，支持复杂参数校验               |
| `service/`     | 核心业务逻辑层，纯Python函数，无HTTP/数据库依赖，易测试、易复用         |
| `tests/`       | 分层测试，覆盖接口层和业务层，保障代码变更不破坏原有功能                 |

## 四、生产级修改升级（核心优化）
### 1. 升级后核心文件
#### （1）接口层：`app/api/v1/user.py`
```python
# 1. 导入扩展依赖：依赖注入、密码加密、自定义异常、业务层
from fastapi import APIRouter, HTTPException, Depends
from pydantic import BaseModel
from passlib.context import CryptContext  # 密码加密工具
from app.core.config import settings      # 全局配置
from app.core.exceptions import BusinessException  # 自定义业务异常
from app.db.mysql import get_db_session   # 数据库会话依赖注入
from app.models.user import User
from app.schemas.user import UserRegisterRequest, UserRegisterResponse  # 独立校验模型
from app.service.user_service import user_register_service  # 业务逻辑解耦
from sqlalchemy.orm import Session

# 2. 初始化路由（保持前缀/标签不变）
router = APIRouter(prefix="/v1/user", tags=["用户管理"])

# 3. 密码加密上下文：指定bcrypt算法，自动加盐，兼容旧算法废弃
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# 4. 注册接口：解耦+安全+异常处理+响应模型约束
@router.post("/register", response_model=UserRegisterResponse)  # 强约束响应格式
def user_register(
    req: UserRegisterRequest,
    db: Session = Depends(get_db_session)  # 依赖注入：自动创建/关闭会话
):
    try:
        # 4.1 调用业务层：接口层仅处理HTTP逻辑，业务逻辑全部下沉
        user_id = user_register_service(db, req)
        # 4.2 标准化响应：符合响应模型约束
        return {"code": 200, "msg": "注册成功", "data": {"user_id": user_id}}
    except BusinessException as e:
        # 4.3 捕获业务异常：如邮箱/用户名已存在
        raise HTTPException(status_code=e.code, detail=e.msg)
    except Exception as e:
        # 4.4 捕获系统异常：屏蔽底层错误，返回通用提示（生产级需加日志）
        raise HTTPException(status_code=500, detail="服务器内部错误")
```

#### （2）校验层：`app/schemas/user.py`
```python
# 1. 导入Pydantic核心模块：字段校验、模型基类
from pydantic import BaseModel, EmailStr, field_validator

# 2. 注册请求校验模型：强化参数校验规则
class UserRegisterRequest(BaseModel):
    username: str
    email: EmailStr
    password: str

    # 2.1 自定义密码校验：长度≥8，包含字母+数字
    @field_validator("password")
    def validate_password(cls, v):
        if len(v) < 8:
            raise ValueError("密码长度不能少于8位")
        if not any(char.isdigit() for char in v) or not any(char.isalpha() for char in v):
            raise ValueError("密码必须包含字母和数字")
        return v

# 3. 注册响应模型：强约束返回格式，避免前端解析异常
class UserRegisterResponse(BaseModel):
    code: int
    msg: str
    data: dict
```

#### （3）业务层：`app/service/user_service.py`
```python
# 1. 导入依赖：数据库会话、模型、校验模型、自定义异常、密码加密
from sqlalchemy.orm import Session
from app.models.user import User
from app.schemas.user import UserRegisterRequest
from app.core.exceptions import BusinessException
from passlib.context import CryptContext

# 2. 密码加密上下文（与接口层一致，可抽离到core/utils.py）
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# 3. 密码加密工具函数：封装加密逻辑，便于复用
def hash_password(password: str) -> str:
    return pwd_context.hash(password)

# 4. 注册核心业务逻辑：纯函数，无HTTP依赖，易测试
def user_register_service(db: Session, req: UserRegisterRequest) -> int:
    # 4.1 扩展校验：邮箱+用户名双重查重
    if db.query(User).filter(User.email == req.email).first():
        raise BusinessException(code=400, msg="该邮箱已注册")
    if db.query(User).filter(User.username == req.username).first():
        raise BusinessException(code=400, msg="该用户名已被占用")
    
    # 4.2 密码加密：核心安全升级，杜绝明文存储
    hashed_pwd = hash_password(req.password)
    
    # 4.3 创建用户实例：存储加密后的密码
    new_user = User(
        username=req.username,
        email=req.email,
        password=hashed_pwd
    )
    # 4.4 数据库操作：提交+刷新，确保获取自增ID
    db.add(new_user)
    db.commit()
    db.refresh(new_user)  # 刷新实例，获取数据库自增的user_id
    
    # 4.5 返回用户ID：供接口层组装响应
    return new_user.id
```

#### （4）全局异常：`app/core/exceptions.py`

```python
# 自定义业务异常：区分业务错误与系统错误，便于前端处理
class BusinessException(Exception):
    def __init__(self, code: int, msg: str):
        self.code = code  # 异常状态码
        self.msg = msg    # 异常提示信息
        super().__init__(msg)  # 继承父类异常
```

### 2. 升级代码逐行说明（核心差异点）
| 升级文件                | 代码段                          | 核心作用                                                                 |
|-------------------------|---------------------------------|--------------------------------------------------------------------------|
| `app/api/v1/user.py`    | `Depends(get_db_session)`       | 依赖注入管理数据库会话，自动创建/关闭，避免连接泄露                       |
| `app/api/v1/user.py`    | `response_model=UserRegisterResponse` | 强约束响应格式，前端无需适配多格式返回                                   |
| `app/api/v1/user.py`    | `try/except BusinessException`  | 捕获业务异常，返回精准提示；捕获系统异常，屏蔽底层错误                   |
| `app/schemas/user.py`   | `@field_validator("password")`  | 自定义密码校验规则，提升密码安全性                                       |
| `app/service/user_service.py` | `hash_password()` | 封装密码加密逻辑，便于登录/重置密码等场景复用                           |
| `app/service/user_service.py` | `db.refresh(new_user)` | 刷新ORM实例，确保获取数据库自增的user_id（基础版可能遗漏）|
| `app/core/exceptions.py` | `BusinessException` | 自定义业务异常，区分「邮箱已注册」和「服务器错误」，前端可精准处理        |

### 3. 核心升级点总结
| 优化维度       | 基础版问题                | 升级方案                                                                 |
|----------------|---------------------------|--------------------------------------------------------------------------|
| 安全           | 密码明文存储              | 使用bcrypt加密+自动加盐，存储加密后密码，杜绝泄露风险                     |
| 代码解耦       | 接口层混杂业务逻辑        | 抽离service层，接口层仅处理HTTP，业务逻辑纯函数化，易测试、易扩展         |
| 数据校验       | 仅基础类型校验            | 自定义字段校验（密码复杂度），拆分请求/响应模型，强约束数据格式          |
| 异常处理       | 无统一异常体系            | 自定义业务异常，区分业务错误/系统错误，前端提示更精准                     |
| 数据库管理     | 会话手动管理              | 依赖注入自动管理会话生命周期，避免连接泄露/未关闭问题                     |
| 可维护性       | 响应格式不统一            | 响应模型强约束，所有接口返回格式一致，降低前端对接成本                   |

## 六、使用说明
1. 依赖安装：将以下内容写入`requirements.txt`，执行`pip install -r requirements.txt`
```
fastapi==0.104.1
uvicorn==0.24.0
pydantic==2.4.2
sqlalchemy==2.0.23
passlib==1.7.4
bcrypt==4.0.1
python-dotenv==1.0.0
```
2. 启动项目：在`main.py`中注册路由，执行`uvicorn main:app --reload`
```python
# main.py
from fastapi import FastAPI
from app.api.v1 import user

app = FastAPI(title="用户管理系统", version="1.0")
app.include_router(user.router)

if __name__ == "__main__":
    import uvicorn
    uvicorn.run("main:app", host="0.0.0.0", port=8000, reload=True)
```
3. 接口调试：访问`http://127.0.0.1:8000/docs`，使用FastAPI原生Swagger文档调试注册接口。
